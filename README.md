=========== REST-подобное API для покупки билетов в кино ============

Framework: Yii, v1.1.14
BD: mysql

@author		baster
@author mail	g-serg88@yandex.ru


для установки демо данных необходимо создать бд cinema, 
скопировать и выполнить запросы из файла /cinema/protected/data/cinema.sql
указать пользователя и пароль в файле /cinema/protected/config/main.php

-----------------------------------------------------------------------------------------------------

Просмотр расписание кинотеатра, с возможностью фильтрации по залу:

/cinema?id=1&hall=1

// id - обязательный параметр (id кинотеатра), так же есть возможность указать 
	вместо id кинотеатр его назнавние (параметр меняется на name)
	или указать alias (название на литинице)
// hall - не обязательный параметр, указывает для какого зала вывести расписание

Результат выполнения: ассоциативный массив с расписаниями фильмов в залах кинотератра в формате JSON
-----------------------------------------------------------------------------------------------------

Чтобы просмотреть в каких кинотеатрах/залах идёт конкретный фильм:

/cinema/film?name=Star%20Trek

// id - обязательный параметр (id фиьма), так же есть возможность указать 
	вместо id фильма его назнавние (параметр меняется на name)

Результат выполнения: ассоциативный массив с расписанием на конкретный фильм в формате JSON
-----------------------------------------------------------------------------------------------------

Чтобы проверить, какие места свободны на конкретный сеанс:

/cinema/film/places?id=1_4_5_15


// id - обязательный параметр (составной, состоит из id кинотеатра, id холла, id фильма, id расписание), 
	берется как id любого ответа из вышеописанных функци

Результат выполнения: ассоциативный массив со свободными местами для конкретного сеанса в формате JSON 
-----------------------------------------------------------------------------------------------------

Для покупки билета:

/cinema/ticket/buy?id=20&places=1,2,3,4&row=5

// id - обязательный параметр (id расписания), можно взять из любого массива вышеописанного функционала (seduleId)
// places - обязательный параметр - покупаемые места. можно указать несколько мест через запятую.
// row - обязательный параметр - номер ряда.

Результат выполнения: конкатинация md5 хешей (от id расписания, id кинотеатра, id холла, id фильма, id места и соли 
(текущее время в секундах)) через нижнее подчеркивание (_) зашифрованная алгоритмом base64. Возвращается в формате JSON
-----------------------------------------------------------------------------------------------------

Чтобы отменить покупку, но не раньше, чем за час до начала сеанса:

/cinema/ticket/reject?code=Njg4ZDAwZTRiNzM2N2Y1NzRhYjRlODZkODNiYWQxZTJfXzA3NjNkZjY3NjFmOWFjZmIxYTY3MGE5ZjVlMWEyYjRjX19mNDM1NGU0Mzk2ODYxOTdiZTgxYzAyZTFmNWNiNTczNF9fMTQ4MjBkODdjZTk4ZTYxZjQ2Y2RhZGRiZGJjMGE0ZTU=

// code - обязательный параметр - уникальный код купленный билетов, берется из ответа предыдущей функции.


Результат выполнения: сообщение об откзае от купленных билетов с их количеством, 
либо сообщение о не возможности возврата билетов. Возвращается в формате JSON
